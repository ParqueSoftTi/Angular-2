En este codelab, te presentamos las 5 opciones de proyectos propuestos para el proyecto final de Angular 2, cada alumno debe seleccionar uno y durante de la clase, deben desarrollar un diagrama entidad relación en la plataforma https://www.drawio.com/

El proyecto se desarrollará durante las siguientes sesiones.

Interfaz para el administrador de una biblioteca que le permita:

  • Crear, editar, buscar y borrar libros.
  • Ver la cantidad de libros disponibles de cada ejemplar.
  • Crear, editar, buscar y borrar usuarios.
  • Prestar un libro a un usuario.
  • Recibir libros de regreso de un usuario.

Cada libro debe contar con los siguientes datos:

  • Título
  • Autor
  • Año de publicación
  • Género
  • Cantidad disponible
  • Código ISBN

Interfaz de usuario para administración básica de un parqueadero que permita:

  • Ingresar un vehículo al parqueadero tomando la hora de ingreso en el sistema, y el tipo de vehículo que ingresa.
  • Registrar la salida con la hora del sistema y realizar el cobro.
  • Llevar un control de los ingresos (caja).
  • La caja se podrá vaciar y se debe llevar un control de la fecha y hora de los vaciados de caja.

El sistema debe permitir que se configuren los siguientes parámetros:

  • Nombre del parqueadero
  • Valor de hora o fracción para carros y motos
  • Cantidad de minutos de gracia (permitiendo el 0)

Programa para calcular los gastos aproximados de un viaje en vehículo. Se espera que el usuario tenga una interfaz para introducir las siguientes variables:

  • Distancia en kilómetros del destino.
  • Cantidad de kilómetros por galón de combustible.
  • Costo del galón de combustible.
  • Costo de cada peaje.

Una vez ingresados estos valores, el sistema debe arrojar un costo aproximado del viaje, incluyendo el tiempo en horas suponiendo una velocidad constante de 60Km/h. Si el viaje dura más de 12 horas, se le debe advertir al usuario que descanse durante el viaje.

Programa para calcular los gastos de los consumibles de un vehículo. Se espera que el usuario pueda ingresar la hoja de vida de su vehículo con las siguientes variables:

  • Kilometraje de inicio.
  • Kilometraje del último cambio de aceite.
  • Kilometraje del último cambio de frenos.
  • Kilometraje del último mantenimiento preventivo.
  • Kilometraje del último cambio en filtro de aceite.
  • Kilometraje del último cambio en el filtro de aire.

El sistema deberá permitir que se configure cada cuantos kilómetros se desea cambiar: frenos, aceite, mantenimiento preventivo, filtro de aire y filtro de aceite.

Se espera que el sistema presente una interfaz con el resumen del kilometraje faltante para los cambios en los consumibles configurados, y deberá permitir la fácil actualización del kilometraje actual del vehículo.

Usar la aplicación ya desarrollada del carrito de compras para ofrecer persistencia en el uso, es decir, que el carrito de compras se almacene en el computador del usuario para que cada que se abra la app, esté la información del carrito en donde se había dejado.
